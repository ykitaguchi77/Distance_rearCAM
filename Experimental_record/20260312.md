# 2026-03-12 実験記録
## フロントカメラ（インカメラ）切り替え機能の追加

### 概要 (Overview)
撮影画面にカメラ切り替えボタンを追加し、フロントカメラ（TrueDepthカメラ）での自撮り撮影に対応した。
フロントカメラ使用時は `ARFaceTrackingConfiguration` を使い、`ARFaceAnchor` から顔までの距離を取得する。

### 変更元ファイル (Source file)
- `ARSessionManager.swift`
- `CameraView.swift`
- `ContentView.swift`

### Before / After 比較
| 項目 | Before | After |
|------|--------|-------|
| カメラ | リアカメラ（LiDAR）のみ | リア + フロント切り替え可能 |
| 距離取得方法 | LiDAR depthMapのみ | リア: depthMap / フロント: ARFaceAnchor |
| AR設定 | ARWorldTrackingConfigurationのみ | リア: WorldTracking / フロント: FaceTracking |
| 起動条件 | LiDAR必須 | LiDARまたはTrueDepthカメラがあれば起動可能 |
| UIボタン | フラッシュ + ズーム + 撮影 | カメラ切替 + フラッシュ + ズーム + 撮影 |
| フロントカメラ映像 | - | ミラーリング表示（自撮りとして自然な表示） |

### 変更理由 (Reason for change)
- 自撮りでの眼瞼距離測定ニーズに対応
- TrueDepthカメラのARFaceAnchorで顔距離が取得可能

### 技術メモ
- `ARFaceTrackingConfiguration` はTrueDepthカメラ搭載デバイスで使用可能
- `ARFaceAnchor.transform.columns.3` のxyz距離（ユークリッド距離）で顔までの距離を算出
- フロントカメラ映像は `scaleEffect(x: -1)` で左右反転（ミラーリング）表示
- フロントカメラ時はフラッシュ非対応（自動無効化）

### ハマった点と解決策 (Troubleshooting)

#### 1. カメラ切替時にフリーズ・リアに戻る問題
- **症状**: インカメ切替ボタンを押すとフリーズし、`FigCaptureSourceRemote err=-12784`（カメラリソース競合）が大量発生
- **原因**: ARSCNViewにtransform（`CGAffineTransform`, `CATransform3D`, SwiftUI `scaleEffect`）を適用すると、Metal描画パイプラインが壊れてセッションが失敗する
- **解決**: ARSCNViewおよびそのラッパーに一切のtransformをかけない。ミラー表示は不要と判断
- **補足**: `arSession.pause()` → `arSession.run(newConfig)` でカメラ切替自体は問題なく動作する。delayも不要

#### 2. ARViewContainer（UIViewRepresentable）の再生成問題
- **問題**: structのプロパティが変わるとSwiftUIがdismantle→makeUIViewを呼び、セッションが停止される
- **解決**: 変化するパラメータをstructに持たせず、ラッパーUIViewの子ビューとしてARSCNViewを配置。sessionManagerは参照型なので構造変更なし

#### 3. Vision orientation
- **結論**: フロント/リア共通で同じorientation（`.right`等）を使用。ミラー付きorientation（`.leftMirrored`等）は不要
