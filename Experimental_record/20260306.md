# 2026-03-06 実験記録

## SegFormer眼瞼セグメンテーション解析モード追加

### 概要 (Overview)
カメラボタン押下時にYOLO検出済みの眼瞼ROIを切り出し、SegFormer B3モデルで眼瞼・虹彩・瞳孔のセグメンテーション解析を行う静止画解析モードを追加。

### 変更元ファイル / 新規ファイル

| ファイル | 種別 | 内容 |
|---------|------|------|
| `convert_segformer.py` | 新規 | PyTorch→CoreML変換スクリプト |
| `ARSessionManager.swift` | 改修 | EyeROI/CapturedFrame追加、captureFrameForAnalysis()、resumeSession() |
| `SegmentationManager.swift` | 新規 | CoreML推論エンジン（前処理・推論・後処理・オーバーレイ生成） |
| `AnalysisView.swift` | 新規 | 解析結果表示UI（ScrollView、EyeResultCard、凡例） |
| `CameraView.swift` | 改修 | ボタンアクション変更、fullScreenCover追加、セッション管理 |

### Before / After 比較

| 項目 | Before | After |
|------|--------|-------|
| カメラボタン動作 | スクリーンショット保存 | ROI切出し→SegFormer解析→結果表示 |
| 解析画面 | なし | fullScreenCoverで解析結果表示 |
| ARSession管理 | pause/start のみ | pause/resume/start（解析中は一時停止） |
| データモデル | FaceData のみ | FaceData + EyeROI + CapturedFrame + SegmentationResult |
| ML推論 | YOLO（リアルタイム） | YOLO（リアルタイム） + SegFormer（静止画） |

### 変更理由 (Reason for change)
- YOLO検出はbbox検出のみ。眼瞼・虹彩・瞳孔の形状解析にはピクセルレベルのセグメンテーションが必要
- SegFormer B3は高精度なセグメンテーションモデルだが567MBと大きく、リアルタイム推論には不向き→静止画解析モードとして実装

### 設計上の重要決定
1. **遅延読込**: 567MBモデルは初回解析時に読み込み（起動時間への影響回避）
2. **cpuAndNeuralEngine**: ARKit Metal競合回避のためGPU回避
3. **逐次処理**: 右眼・左眼を逐次処理（メモリ節約）
4. **dataPointer高速アクセス**: MLMultiArrayのNSNumberサブスクリプトは遅いため、Float32ポインタで直接アクセス
5. **premultiplied alpha**: iOSの描画系に合わせた正しいアルファ処理
6. **ROI正方形化**: bbox幅×1.5→正方形にすることで、512×512入力に歪みなく渡す

## カメラ操作UI追加（フラッシュ・ズーム・レイアウト改善）

### 概要 (Overview)
CameraViewのUI改善: 撮影ボタン右下移動、フラッシュON/OFF（撮影時のみ一瞬点灯）、倍率切替（×1/×2/×3）、推論中の撮影画像表示

### 変更元ファイル

| ファイル | 内容 |
|---------|------|
| `ARSessionManager.swift` | `import AVFoundation`追加、フラッシュ/ズーム制御メソッド追加 |
| `CameraView.swift` | UIレイアウト再構成（3カラム配置） |
| `AnalysisView.swift` | 推論中に撮影画像を背景表示 |

### Before / After 比較

| 項目 | Before | After |
|------|--------|-------|
| 撮影ボタン位置 | 中央下 | 右下 |
| フラッシュ制御 | なし | 左中央に⚡ボタン（撮影時のみ一瞬torch点灯） |
| ズーム制御 | なし | 左下に×1/×2/×3ボタン（選択中はハイライト） |
| 推論中表示 | ProgressViewのみ | 撮影画像 + ultraThinMaterial + ProgressView |
| キャプチャ方式 | 同期 | 非同期（フラッシュ時0.2s待機後スナップショット） |
| セッション復帰 | 設定なし | ズームファクター復元 |

### 変更理由 (Reason for change)
- 撮影時のライティング不足をフラッシュで補完（特に眼瞼撮影では照明が重要）
- ズーム機能で近距離撮影の補助（広角レンズで小さな被写体を拡大）
- 推論中に「何を撮影したか」をユーザーが確認できるようにUX改善

### ハマった点と解決策 (Troubleshooting)
- **非同期キャプチャ問題**: フラッシュ対応で `captureFrameForAnalysis()` が `asyncAfter` を使う非同期処理になったため、呼び出し直後に `capturedFrame` を参照できなくなった → `captureCount` カウンターを追加し `onChange` で監視する方式に変更

## 撮影画面4項目修正（ズーム・プレビューフロー・ぼやけ・高画質キャプチャ）

### 概要 (Overview)
撮影画面の4つの問題を修正:
1. ズームボタンが効かない→ARKit対応のレンズ切替+デジタルズーム方式に変更
2. 撮影→即解析→プレビュー→解析ボタンのフローに変更
3. 解析中の写真ぼやけ→ブラーマテリアル除去
4. 高画質キャプチャ（captureHighResolutionFrame）対応追加

### 変更元ファイル

| ファイル | 内容 |
|---------|------|
| `ARSessionManager.swift` | ズーム方式全面変更、高解像度キャプチャ追加、キャプチャ処理リファクタ |
| `AnalysisView.swift` | プレビューフロー導入、ぼやけ修正 |
| `CameraView.swift` | HD切替ボタン追加 |

### Before / After 比較

| 項目 | Before | After |
|------|--------|-------|
| ズーム方式 | AVCaptureDevice.videoZoomFactor（ARKit下で無効） | レンズ切替(望遠フォーマット) + CGAffineTransformデジタルズーム |
| 撮影フロー | 撮影→即解析開始 | 撮影→プレビュー(全体画像+ROIサムネイル)→解析ボタン→解析 |
| 解析中背景 | `.ultraThinMaterial`（ブラー効果でぼやける） | `Color.black.opacity(0.3)`（クリア表示） |
| 推論中テキスト色 | `.secondary`（薄いグレー） | `.white`（黒背景に映える） |
| キャプチャ方式 | snapshot()のみ | captureHighResolutionFrame（デフォルト） + snapshot（フォールバック/切替可能） |
| HDトグル | なし | 左サイドに「HD」ボタンで切替 |
| セッション設定 | videoHDRなし | `config.videoHDRAllowed = true` |

### 変更理由 (Reason for change)
- **ズーム**: ARKitがAVCaptureSessionを専有するため、直接AVCaptureDeviceのzoomFactorを変更しても反映されない。`supportedVideoFormats`から望遠レンズフォーマットを選択しセッション再起動することで光学ズーム相当を実現
- **プレビューフロー**: ユーザーが撮影結果とROI切り出しを確認してから解析開始できるようにUX改善。不要な解析の実行を防止
- **ぼやけ修正**: `ultraThinMaterial`はiOSのビジュアルエフェクトで背景にガウシアンブラーをかけるため、解析中の撮影確認に不適切
- **高画質キャプチャ**: snapshot()はビュー解像度依存で低画質。captureHighResolutionFrameは最大12MPの高解像度フレームを取得可能

### 設計上の重要決定
1. **2段階ズーム**: x1=広角(デフォルト)、x2=望遠レンズ切替、x3=望遠+1.5xデジタルズーム
2. **デジタルズームはtransformベース**: ARSCNView.transformにCGAffineTransformを適用。キャプチャ時はapplyZoomCrop()で中央クロップ
3. **captureHighResolutionFrameのフォールバック**: 失敗時はsnapshot()にフォールバック
4. **CVPixelBuffer→UIImage変換**: CIImage経由で変換、landscape-right→portrait回転（.right orientation）
5. **buildCapturedFrame共通化**: snapshot/highResどちらのパスでも同じROI切り出しロジックを使用

### 注意点
- デジタルズーム中はFaceOverlayViewとカメラ表示のずれが生じる可能性あり（transformはARSCNViewのみに適用）

## 撮影画面簡素化（レンズ切替廃止・キャンセル機能追加）

### 概要 (Overview)
- レンズ切替方式のズームがうまく動作しなかったため、純粋なデジタルズーム（CGAffineTransform + クロップ）のみに簡素化
- captureHighResolutionFrame を廃止し、シンプルなsnapshot()に統一
- 解析中に「動画に戻る」を押した時にinferenceをキャンセルして撮影に戻れるようにした

### 変更元ファイル

| ファイル | 内容 |
|---------|------|
| `ARSessionManager.swift` | レンズ切替・highResCapture削除、デジタルズームのみに |
| `SegmentationManager.swift` | `cancelled`フラグ + `cancel()`メソッド追加 |
| `AnalysisView.swift` | 解析中の「動画に戻る」で`cancel()`呼び出し |
| `CameraView.swift` | HDトグルボタン削除 |

### Before / After 比較

| 項目 | Before | After |
|------|--------|-------|
| ズーム方式 | レンズ切替 + デジタルズーム | デジタルズームのみ（transform + クロップ） |
| キャプチャ | captureHighResolutionFrame / snapshot切替 | snapshot()のみ |
| 解析中キャンセル | 不可（画面を閉じても推論継続） | cancel()でROIループを中断、結果を破棄 |
| UIボタン | フラッシュ + HD + ズーム | フラッシュ + ズーム |

### ハマった点と解決策 (Troubleshooting)
- **レンズ切替が効かない**: `supportedVideoFormats`から望遠フォーマットを選択してセッション再起動しても倍率変化が見られなかった → デジタルズームのみに簡素化
- **UIView.transformがSwiftUIで効かない**: `ARSCNView.transform = CGAffineTransform(scaleX:y:)`を設定しても、SwiftUIの`UIViewRepresentable`がレイアウト更新時に上書きし反映されない → SwiftUIの`.scaleEffect()`に変更して解決
- **ズーム時にROIがズレる**: `snapshot()`はフルフレームを返すが、ズームクロップ後の画像に対してフルフレーム基準のface座標でROIを切り出していた → ROI切り出しはフルフレームから行い、表示用fullImageのみクロップする方式に分離して解決
